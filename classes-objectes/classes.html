Classes i objectes

Pensa en coses que la gent veu cada dia, com cotxes, botigues i aus. Tots aquests són objectes: coses tangibles amb què la gent pot observar i interactuar.

Quines són algunes de les qualitats d'aquests objectes? Un cotxe té rodes. Les botigues venen articles. Les aus tenen ales.

Aquestes qualitats, o propietats, defineixen els que constitueix un objecte. Tingues en compte que objectes similars comparteixen les mateixes propietats, però possiblement tinguin valors diferents per a aquestes propietats. Per exemple, tots els cotxes tenen rodes, però no tots els cotxes tenen la mateixa quantitat de rodes.

Els objectes en JavaScript són usats per modelar objectes de el món real, donant-los propietats i comportaments com els seus contraparts de el món real. Aquí hi ha un exemple usant aquests conceptes per crear un objecte duck (ànec):


    let duck = {
    name: "Aflac",
    numLegs: 2
    };
    console.log(duck.name)

L'objecte duck té dos parells propietat / valor: 1 name (nom) de Aflac i un numLegs (nombre de potes) de 2.

Crea un objecte dog amb les propietats name i numLegs i assigna'ls una cadena i un nombre, respectivament.


    let dog = {
    name: "Spot",
    numLegs: 4
    };
    console.log(dog.name)

Els objectes poden tenir un tipus de propietat, anomenada mètode.

Els mètodes són propietats que són funcions. Aquests afegeixen diferents comportaments als objectes. Aquí aquesta l'exemple d'duck amb un Metode


let duck = {
  name: "Aflac",
  numLegs: 2,
  sayName: function() {return "The name of this duck is " + duck.name + ".";}
};
duck.sayName();

Aquest exemple afegeix el mètode sayName, el qual és una funció que retorna una oració que lliura el nom de l'duck (ànec). Tingues en compte que el mètode va accedir a la propietat name en la sentència de retorn usant duck.name. El següent desafiament abastés una altra manera de fer això.

Usant l'objecte dog, assigna-li un mètode anomenat sayLegs. El mètode ha de retornar la frase This dog has four legs.


let dog = {
  name: "Spot",
  numLegs: 4,
  sayLegs: function() {
    return "This dog has " + dog.numLegs + " legs.";
  }
};

dog.sayLegs();




let dog = {
  name: "Spot",
  numLegs: 4,
  sayLegs: function() {
    return "This dog has " + this.numLegs + " legs.";
  }
};

dog.sayLegs();

Tot i que aquesta és una forma vàlida d'accedir a la propietat de l'objecte, hi ha un problema. Si el nom de la variable canvia, qualsevol codi que faci referència a el nom original també hauria de ser actualitzat. En una definició breu d'un objecte, això no és un problema, però si un objecte té moltes referències a les seves propietats hi ha una major probabilitat d'error.

Una manera d'evitar aquests problemes és amb paraula clau this:

this és un tema profund, i l'exemple anterior és només una forma d'usar-lo. En el context actual, this es refereix a l'objecte amb el qual el mètode està associat: duck. Si el nom de l'objecte es canvia a Mallard, no cal trobar totes les referències a duck en el codi. Fa que el codi sigui reutilitzable i mes fàcil de llegir.

Funció "Constructor"

Les funcions Constructors creen nous objectes. Defineixen propietats i comportaments que pertanyen a el nou objecte. Pensa que són el model per a la creació de nous objectes.

A continuació es mostra un exemple d'un constructor:



function Bird() {
  this.name = "Albert";
  this.color = "blue";
  this.numLegs = 2;
}

Aquest constructor defineix un objecte Bird amb les propietats name, color i numLegs establertes, Albert, blue i 2 respectivament. Els constructors tenen les següents convencions:

 Estan definits amb un nom en majúscula per distingir-los d'altres funcions que no són constructors.

 Utilitzen la paraula clau this per establir propietats de l'objecte que crearan. Dins el constructor, this es refereix a el nou objecte que crearà.

 Els Constructors defineixen propietats i comportaments.
